Making a site responsive is the new de-facto standard of the industry. No one is spared, not even the WordPress developers. Let's see what we can do here. Will be talking about the logic that goes behind making responsive designs, some standard practices, code snippets, ideas etc.

the Logic behind Responsive Design

  1. Layouts & Breakpoints Decide your break-points, popular breakpoints for devices are available online. Common practice is to keep 2-3 columns layout for desktop. 1-2 column for iPad and 1 column for phones.
  2. Work on Readability & Typography: This is more important, test it out on devices and adjust accordingly.
  3. Images: Optimize for connections and Retina Display.

the Code behind Responsive Design

  1. Crash Course in CSS
  2. Quick overview on @media queries and view port settings.
  3. Use a framework like inuit.css, Zurb Foundation etc.

Tricks of Trade for Responsive Design

  1. Navigation: hidden and full-glory navigations. If it's too long, hide it!
  2. Kill images if they are not important if they are not crucial.
  3. Don't just resize your browser, test it on actual devices.

Pornography for Developers: Some ideas

  1. Fix social media buttons on mobile phones.
  2. Ideas on Responsive Options for Theme Back-end.
